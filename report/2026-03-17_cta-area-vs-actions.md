# 리서치 리포트: "CTA Area" vs "Actions" — 바이브코딩 핏 비교

> 생성일: 2026-03-17 | 신뢰도: 96% | 소스: 25개 | 라운드: 1/3

## 배경

TDS naming-policy v1.2 개정 중, 래퍼 프레임 접미사를 Area로 통일하는 과정에서 `Container CTA` → `CTA Area` vs `Actions` 중 어느 쪽이 바이브코딩(AI 코드 생성)에 더 적합한지 검증 필요.

오늘 세션에서 합의된 래퍼 네이밍 우선순위:
1. 역할명만 (Header, Footer)
2. 복수형 (Actions, Buttons) — 이번 리서치 대상
3. Area (최후 수단)

## Executive Summary

**`Actions`가 압도적으로 우수.** CTA는 마케팅 도메인 용어로 AI가 독립 마케팅 섹션(hero+큰 버튼)으로 오역할 리스크가 있고, shadcn/Radix/MUI/Ant Design 어디서도 컴포넌트명으로 사용하지 않음. Actions는 모든 주요 디자인 시스템의 사실상 표준 패턴명.

## 핵심 발견

### 1. CTA는 shadcn/Radix에서 0건 사용

shadcn/ui 전체 컴포넌트(59개+)와 Radix Primitives 전체를 조사한 결과, "CTA"라는 컴포넌트명/파트명/클래스명은 **단 한 건도 없음**. CTA는 오직 서드파티 블록 라이브러리(Shadcnblocks, Launch UI)에서 마케팅 섹션 블록명으로만 사용됨.

> 소스: shadcn/ui 공식 문서 전수 조사 (A) ✅ WebFetch

### 2. Actions는 업계 사실상 표준

| 디자인 시스템 | 사용 방식 | 컴포넌트명 |
|---|---|---|
| MUI | Card, Dialog 하위 컴포넌트 | `CardActions`, `DialogActions` |
| Ant Design | Card prop | `actions: ReactNode[]` |
| shadcn/Radix | 개별 액션 버튼 + 그룹 | `AlertDialogAction`, `CardAction`, `ItemActions` |
| Chakra UI | Footer 설명에서 | CardFooter "houses the card **actions**" |
| Angular Material | Card 하위 | `mat-card-actions` |

> 소스: MUI (A), Ant Design (A), shadcn (A), Chakra (A) ✅ 교차 검증

### 3. CTA = 마케팅 용어, Actions = 구조적 컨테이너

- **CTA**: "Call to Action" — 버튼의 시각적 강조 수준 (what kind of button)
- **Actions**: 인터랙티브 요소 그룹을 담는 컨테이너 (where buttons go)

Tailwind UI에서 CTA는 `/marketing/sections/cta-sections` 카테고리에 분류됨.

> 소스: Tailwind UI (A), Nathan Curtis Slots in DS (B) ✅ WebFetch

### 4. AI 코드 생성 시 해석 차이

| 레이어명 | AI 생성 코드 | 리스크 |
|---------|-------------|--------|
| `CTA Area` | `<section className="cta-section">` + 헤딩 + 큰 버튼 | 마케팅 블록으로 오해 |
| `Actions` | `<div className="actions">` + 버튼 그룹 / `<CardActions>` | 거의 없음 |

Figma MCP 공식 가이드: "Name layers semantically" — 코드 세계에서 통용되는 이름일수록 AI 매핑 정확도 상승.

> 소스: Figma MCP Server Guide (A), bolt.new Guide (B), Figmentor (B) ✅ WebFetch

### 5. shadcn에서 Action vs Actions vs Footer 구분

| 패턴 | 의미 | 예시 |
|------|------|------|
| `*Action` (단수) | 단일 액션 버튼 | `AlertDialogAction`, `CardAction` |
| `*Actions` (복수) | 여러 버튼의 컨테이너 | `ItemActions` |
| `*Footer` | 하단 영역 (액션 포함 가능) | `DialogFooter`, `CardFooter`, `DrawerFooter` |

TDS에서 버튼 그룹 래퍼에는 복수형 `Actions`가 가장 정확.
