## Phase 1: 전문가별 분석

### Design Director

**핵심 팩트:** Inter에는 한글(Hangul) 글리프가 없다. Latin/Cyrillic/Greek 전용.

| 항목 | Pretendard (현재) | Inter (전환 시) |
|------|-------------------|----------------|
| Latin 렌더링 | ✅ Inter 동일 | ✅ 동일 |
| 한글 렌더링 | ✅ 네이티브 (통일된 디자인) | ❌ 시스템 fallback (불일치) |
| Latin+한글 혼합 | ✅ 균일한 x-height, weight | ❌ 폰트 간 weight/baseline 불일치 |
| Figma에서 한글 | ✅ 단일 폰트 | ⚠️ fallback 혼재 |
| 프로덕트 코드 | ✅ 단일 폰트 | ❌ font-stack 필요 |

**디자인 시스템 준수 기준:**
- 타이포그래피 통일성: ❌ FAIL — 2개 폰트 혼재
- weight 일관성: ❌ fallback 폰트 weight 매핑 불확실

**입장: No-go (Inter 단독).** 한글 제품에서 한글 폰트 없는 건 불가. 대안으로 Noto Sans KR 제안.

**Pretendard vs Noto Sans KR 디자인 감성:**
- Pretendard: Inter 기반 → Geometric sans-serif, 모던, 깔끔
- Noto Sans KR: Humanist 계열, 구글 표준, 가독성 우수하지만 다른 성격
- → 전환 시 모든 화면의 "느낌"이 달라짐. 3개 핵심 화면 목업 비교 선행 필요.

### Engineering Lead

**마이그레이션 비용 (A 또는 B 진행 시):**

| 작업 | 소요 시간 | 리스크 |
|------|----------|--------|
| TDS Text Style 전체 교체 | 2-4시간 | 낮음 |
| TDS 컴포넌트 폰트 교체 | 2-4시간 | 중간 |
| 프로덕트 디자인 파일 교체 | 1-2시간 | 낮음 |
| 코드 font-family 변경 | 30분 | 낮음 |
| lineHeight/letterSpacing 재조정 | **4-8시간** | **높음** |

**기술 부채 평가:**
- Pretendard → Inter: 새로운 기술 부채 생성 (font-stack 관리, OS별 fallback 테스트)
- Pretendard → Noto Sans KR: 기술 부채 없음 (단일 폰트)

**새 옵션 제안 — MCP 중간 폰트:**
```
MCP use_figma: Noto Sans KR로 생성 (한글 네이티브, MCP 가용)
→ 생성 후: Pretendard'em All 또는 TDS Tools로 일괄 전환
→ TDS 표준은 Pretendard 유지
```

**입장:** Inter 단독은 기술적으로 가능하지만 새 부채 생성. 전환할 거면 Noto Sans KR. 최적은 Pretendard 유지 + 워크어라운드.

### Product Leader

**사용자 문제 중심 분석:**
- 폰트 전환이 해결하는 사용자 문제 = 없음 (개발자 도구 마찰 제거가 목적)
- 폰트 전환이 만드는 사용자 문제 = 한글 타이포 품질 하락

**RICE — Inter 전면 전환:**
- Reach: 전체 사용자 / Impact: -1 (부정적) / Confidence: 90% / Effort: 2주
- → RICE = 부정적

**입장: No-go (Inter 단독).** 사용자 가치를 줄이면서 개발자 편의를 높이는 트레이드오프는 정당화 불가. Noto Sans KR은 추가 검토 가치 있으나, Pretendard 유지 + 워크어라운드 추천.

## Phase 2: 종합

### 합의점
- **Inter 단독 전면 전환은 No-go** — 한글 제품에서 한글 폰트 없는 건 불가 (전원 동의)
- **Pretendard는 TDS 표준으로 유지** — 디자인 감성 + 한글 품질이 핵심 자산
- **MCP 문제는 폰트 교체가 아닌 워크어라운드로 해결** — Figma 로드맵에 이미 있음
