# Already Following Recommendation Source

- **Date**: 2026-05-06
- **Mode**: Team Mode — Trade-off Discussion
- **Roles**: Product Leader, Design Director, Growth Expert
- **Topic**: 이미 팔로우 중인 친구에게 연락처 기반 추천 여부를 구분해 보여줄 필요가 있는가

## Phase 1: Expert Analyses

### Product Leader

Using the user-problem framework, the main user problem after following is no
longer "why should I follow this person?" but "who am I already connected to?"
Recommendation source is useful before the follow decision, but has low
actionability after the user already follows them.

Recommendation: keep the source in data and analytics, but do not make it a
primary row label for already-following friends.

### Design Director

Using UX clarity and information hierarchy criteria, already-following rows
should prioritize relationship state and recognizable identity. Showing
`연락처에서 가져옴` next to already-following users can create confusion because it
sounds like an unresolved recommendation rather than an established connection.

Recommendation: if needed, use a small secondary badge such as `연락처 친구`,
not a recommendation-source sentence.

### Growth Expert

Using Activation Optimization, source labels increase follow conversion for
not-yet-followed recommendations. After conversion, the source label has limited
growth value and can add noise. The system should still retain acquisition
source for attribution and ranking feedback.

Recommendation: source is visible for recommendations; hidden or de-emphasized
after follow.

## Phase 2: Synthesis

### Agreements

- Track source internally for all recommendations.
- Show source prominently only before follow.
- For already-following users, show relationship state (`팔로잉`) and optionally a
  lightweight relationship badge (`연락처 친구`) if it changes user understanding.

### Conflicts

- Product and Growth prefer hiding source after follow to reduce noise.
- Design allows a small badge only when contact-origin context is important to
  the screen's job.

### Recommendation

Do not show "연락처 기반 추천이었는지" as a primary distinction for already-following
friends. Use these display rules:

1. Not followed recommendation: show `연락처에서 가져옴`, `함께한 챌린지 기반`, etc.
2. Already following in recommendation results: either exclude from the
   recommendation list or show disabled state `팔로잉`.
3. Following/friend management: optionally show `연락처 친구` only if contact-sync
   management or privacy explanation matters.

## Phase 3: QA

PASS. The recommendation separates decision-stage recommendation metadata from
post-follow relationship metadata and preserves attribution internally.
