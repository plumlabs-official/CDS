## 핵심 결정

| 결정 | 내용 | 근거 |
|------|------|------|
| 신규 vs 기존 | 기존 TDS 플러그인에 모듈 추가 | 빌드 인프라 재사용, UI 통합 |
| 기존 naming 모듈 | 레거시 유지 (미수정) | 충돌 회피, 공존 |
| AI 서버 | 불필요 (100% 규칙 기반) | naming-policy v1.1이 충분히 구체적 |
| 상수 파일 | 새 rules.ts 생성 (constants.ts 미수정) | 레거시 호환 |

## 신규 파일 구조

```
figma-plugins/tds/src/modules/renamer/
├── renamer.ts           ← renameProductDesign + renameTDSLibrary
├── rules.ts             ← naming-policy v1.1 → 코드
├── property-checker.ts  ← Mode 2 프로퍼티 점검
└── utils.ts             ← isTDSInstance, walkTree, inferContext
```

## 두 모드

| | Mode 1: Product | Mode 2: TDS Library |
|-|-----------------|---------------------|
| 대상 | FRAME/GROUP | COMPONENT/COMPONENT_SET |
| 공식 | [컨텍스트] [역할] | [컴포넌트] [파트] |
| TDS 내부 | skip | 검사 대상 |
| 프로퍼티 | 안 함 | camelCase + show + lowercase |
| Container | 금지 → Content/Area/Group | M3 예외 허용 |

## 기존 코드 충돌점

| 기존 constants.ts | 새 rules.ts | 해소 |
|------------------|-------------|------|
| Container ∈ VALID_SEMANTIC_TYPES | Container 금지 | 별도 파일로 분리 |
| Content ∈ BLACKLIST_PATTERNS | Content = 1순위 허용 | 별도 파일로 분리 |
| 슬래시 네이밍 | 슬래시 = variant 전용 | 새 모듈에서 공백 사용 |
| PascalCase | Title Case 공백 | 새 모듈에서 공백 삽입 |
